- TJLH 公司 3 条配置全部 status=draft，无 published 配置

**修复**：
1. 数据：config id=60 status 改为 published

3. api.py：customer_config 无已发布配置时返回空默认（`{ui:{},fields:[],copy:{}}`），不抛 404

## Supabase 400 修复（P0）

**问题**：Netlify 部署的前端启动时，Supabase Storage bucket `ex` 的 4 个文件（config.json / version.json / price.bundle.json / stock.bundle.json）全部返回 400。

**根因**：
1. `_deploy_bundles_to_supabase()` 只上传 price.bundle.json + stock.bundle.json，**不上传 config.json 和 version.json**
2. `normalize_config()` 不保留 `data_source` 字段 — v2 路径完全丢弃，v3 路径 DEFAULT_CONFIG 没有该字段 → 部署函数第 391 行 `raise ValueError("配置缺少 data_source.base_url")`
3. TJLH published config (id=60) 的 config_json 里没有 `data_source`
4. `quotation_items` 表完全为空 — bundle 即使部署也是 0 行

**修复**：
1. `config.py` DEFAULT_CONFIG 新增 `data_source`（含 base_url/version_file/config_file/price_bundle_file/stock_bundle_file/cache_name）
2. `config.py` `normalize_config()` v2 路径增加 `data_source: deep_merge(DEFAULT_CONFIG["data_source"], raw.get("data_source") or {})`
3. `api.py` `_deploy_bundles_to_supabase()` 重构为上传 4 个文件（price/stock/config/version），提取 `_upload()` 内部函数
4. `api.py` 顶部新增 `from datetime import datetime, timezone` 导入
5. DB: `UPDATE quotation_configs SET config_json = ... WHERE id=60` 注入 data_source

**测试**：58 passed / 5 failed（admin_gui 历史遗留）/ 0 回归

**Bucket 权限修复（关键）**：
- bucket `ex` 是 private（`public: false`），公开 URL 返回 400
- 改用 public bucket `s-q`（`public: true`）
- `apps/app.js` 第 557 行 `SUPABASE_BASE_URL` 从 `/public/ex` 改为 `/public/s-q`
- DB config `data_source.base_url` 同步更新为 `s-q`
- 4 个文件已上传到 `s-q`，全部 HTTP 200

**FastAPI body 参数修复**：
- `from __future__ import annotations` 导致嵌套在 `create_app()` 内的 Pydantic 模型（`BundleGenerate`/`BundleDeploy`）无法被 FastAPI 识别为 body 参数，被当作 query 参数
- 修复：将 `BundleGenerate`/`BundleDeploy` 移到模块级别（与 `CustomerLogin` 同级）
- `POST /api/companies/TJLH/merger/bundle/generate` 现已正常工作，可一键部署 4 文件

**最终状态**：
- 4 个文件在 `s-q` bucket 公开可访问（HTTP 200）
- 后端 `bundle/generate` 端点可用（支持 deploy=true + anon_key）
- 58 passed / 5 failed / 0 回归

**待办**：

## 浏览器缓存导致前端仍请求旧 ex bucket（P0）
