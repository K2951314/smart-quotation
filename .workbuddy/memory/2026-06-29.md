## 对抗式审查 + doCopy / buildStockClipboardLine 根治 + CSS 冲突修复（2026-06-29）

**审查发现的问题**：

1. **`doCopy()` 硬编码字段名**：`enabledFields.c/spec/price/s/i/r` 硬编码了 DEFAULT_COPY_COLUMNS 的 field 值。当远程配置覆盖 copyColumns 使用不同 field 名时（如 `field: "code"` 而非 `"c"`），检查失效。

2. **`normalizeFieldKey("price")` → `"quote_price"` 但行对象实际是 `row.price`**：存在 mapping 偏差隐患。

3. **`buildStockClipboardLine()` 完全不读列复选框**：三菱库存输出硬编码 `[code, spec, price, stock]`，忽略用户勾选的列。

4. **CSS 冲突**：`.toolbar-copy-inline` 有两个裸定义（行1952 grid + 行2689 flex+grid-column:3），后者覆盖前者，破坏 toolbar-secondary 中的 grid 布局。

**修复**：

| 文件 | 改动 |
|---|---|
| `apps/app.js` | 新增 `fieldToRowProp()` 辅助函数，修正 price→quote_price 映射偏差 |
| `apps/app.js` | `doCopy()` 重写为列驱动循环：遍历 `getCopyColumns()`，用 `fieldToRowProp(col.field)` 动态确定行属性名 |
| `apps/app.js` | `buildStockClipboardLine()` 同步列复选框逻辑；库存信息始终追加（不受复选框控制，因为那是查询的核心目的） |
| `apps/styles.css` | `.toolbar-secondary .toolbar-copy-inline` 补齐 `display: grid; grid-template-columns: repeat(6, 1fr); grid-column: auto;`，以高特异性抵御裸选择器覆盖

## doCopy 多行输出 + 发布配置失败根因修复（2026-06-29 10:55）

### Bug 1：doCopy 不按 col.line 分拆多行

**根因**：`doCopy()` 所有列拼成一行 `parts.join(" ")`，完全忽略 `col.line`。而 `buildStockClipboardLine()` 已用 `lineGroup` 正确分拆。

**修复**（`apps/app.js`）：
- `doCopy()` 改为 `mainParts`/`detailParts` 两组
- 按 `col.line || "main"` 分组，main 拼一行，detail 各占一行
- 返回 `[mainParts.join(" ")].concat(detailParts).join("\n")`

### Bug 2：发布配置总是失败（Supabase 同步失败）

**从第一性原理追踪**：
1. 前端 `saveConfig("published")` → `collectConfig()` → `POST /api/companies/TJLH/config` → ✅ 后端保存成功（curl 验证 HTTP 200）
2. 发布后自动部署 Supabase：`sbUploadFile("config.json", ...)` → ❌ **ReferenceError: sbAnonKeyInput is not defined**

**根因**：`sbAnonKeyInput`、`sbBaseUrlInput` 在 `bind()` 内用 `const` 声明；`sbUploadFile()`、`sbGetAnonKey()`、`sbSetStatus()`、`sbAutoFillBaseUrl()` 也全部在 `bind()` 内定义。`saveConfig()` 和 `rollbackToRevision()` 是顶级函数，无法访问 `bind()` 作用域内的变量/函数 → 每次发布都抛出 ReferenceError → 被 catch 捕获 → 显示 "⚠️ Supabase 同步失败"。

**修复**（`admin/app.js`）：
- 新增顶级变量：`let sbAnonKeyInput = null;` `let sbBaseUrlInput = null;`
- 新增顶级常量：`const SB_KEY = "quotation-admin-sb-anon-key";`
- 将 `sbAutoFillBaseUrl()`、`sbSetStatus()`、`sbGetAnonKey()`、`sbGetBaseUrl()`、`sbUploadFile()` 全部提升到顶级作用域
- `bind()` 中改为赋值 `sbAnonKeyInput = $(...)`，删除冗余函数定义
- `$()` helper 改为 `document.getElementById()`（因 sbSetStatus 在顶级，$ 需要在它之前定义）

**重要提示**：即使作用域修复后，Supabase 写入仍需有效的 anon key（在 admin 部署面板填写）。如果 anon key 没有写入权限，上传会报 HTTP 4xx 错误，需配置 Supabase bucket RLS 策略或使用 service_role key。
