## 默认折扣 v3 格式兼容修复（2026-06-08 晚）

### 问题：admin 发布到 Supabase 的默认折扣在主应用不生效
- 根因：admin 管理界面使用 schema v3 格式，折扣规则存于 `config.rules`（含 `when.all`/`actions` 结构）；主应用 `config-core.js` 的 `normalizeDiscountRules()` 只识别 v2 格式 `config.discount_rules`，忽略 v3 `rules` 导致默认折扣回退到内置值 55

### 修复 3 处：
1. **`apps/lib/config-core.js` — `normalizeDiscountRules()`**：新增 v3 `rules` 格式检测与转换（`when.all[].{field,op,value}` → `conditions[].{field,contains/equals/regex/gt/gte/lt/lte}`，`actions[0].percent` → `percent`）
2. **`apps/lib/config-core.js` — `conditionMatches()`**：新增 `gt/gte/lt/lte` 四个数值比较操作符支持
3. **`apps/app.js` — `getRuntimeAppConfig()`**：在调用 `normalizeConfig(cfg)` 前添加 `delete cfg.rules`，防止 v3 原始 rules 覆盖已应用在 discount_rules 上的本地折扣覆盖值

### 验证
- `node -c apps/lib/config-core.js` ✅
- `node -c apps/app.js` ✅
- 端到端模拟测试：v3 rules → normalizeDiscountRules → getDiscountPreset 正确返回 admin 设定的默认折扣值 ✅
- 覆盖值传递测试：getRuntimeAppConfig → delete rules → normalizeConfig 不丢失覆盖 ✅
