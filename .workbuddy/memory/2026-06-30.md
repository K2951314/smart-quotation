## 库存查询「上海库存」误报 Bug 修复

**现象**：WNMG080408-MP MC6125 查询三菱库存，API 返回 `"上海库存189"`（正确），但前端卡片显示「查询失败」。

**根因**：`apps/app.js` `parseStockResultLine()` L1563 错误检测正则 `!/[上日]本?库存/.test(text)` 存在设计缺陷：
- `/[上日]本?库存/` 可匹配 "日本库存"（日→本→库存），但无法匹配 "上海库存"（上 之后是 海，不匹配 本?库存）
- 导致所有仅返回"上海库存"的查询被错误标记为 error

**修复**：将错误检测逻辑改为 `hasStock = (result.shanghai > 0 || result.japan > 0)`，直接基于已解析的数值判断，不再依赖模糊正则。

**修改文件**：`apps/app.js` L1562-1566

**验证**：
- Node.js 模拟测试 5 个场景全部通过
- Railway API 实查 WNMG080408-MP MC6125 + CNMG120408-MA MC5005 均正确解析
- 3 个 JS 测试套件（data-utils / export-utils / config-core）共 11 条全通过
