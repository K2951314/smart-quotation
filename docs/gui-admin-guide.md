# 配置中心操作手册

> 面向非技术人员的 GUI 操作指南。无需修改任何代码或 JSON 文件，所有配置均可在浏览器中完成。

---

## 启动方式

```powershell
# 安装依赖（仅首次）
pip install -r requirements.txt

# 启动服务
py -m backend.smart_quotation
```

浏览器打开：

- 配置中心：`http://127.0.0.1:8001/admin/`
- API 健康检查：`http://127.0.0.1:8001/api/health`

---

## 界面总览

配置中心分三栏：

| 区域 | 说明 |
|------|------|
| 左侧导航栏 | 10 个功能区的快速跳转链接，顶部显示当前已发布版本号 |
| 中间工作区 | 主要操作区域，按导航顺序分为 11 个 Section |
| 右侧预览栏 | 实时预览当前配置 JSON；测试报价（输入关键词即可测试） |

顶栏三个常驻按钮：

| 按钮 | 说明 |
|------|------|
| **读取配置** | 从后端加载当前已发布配置（覆盖页面上的草稿） |
| **保存草稿** | 将当前页面填写的内容保存为草稿，不影响报价台 |
| **发布配置** | 将草稿发布为正式版本，报价台立即使用新配置 |

> **重要**：修改后必须点"发布配置"，报价台才会使用新版本。"保存草稿"只是暂存，不会影响线上运行。

## 快速起步：数据拼接与部署

如果你当前目标是“把文件拼接成价格包 / 库存包并部署到 Supabase”，建议优先使用 `merger/index.html` 页面。该页面入口更清晰、步骤更直接。

快速流程：
1. 进入 `merger/index.html`。
2. 阶段1：选择报价/库存原始文件，点击“分析文件并识别品牌”。
3. 检查识别结果并修正品牌归属。
4. 点击“写入库存”或单个文件导入。
5. 阶段2：加载当前数据后点击“生成 Bundle”，确认正确。
6. 若要同步前端，填写 Supabase Anon Key，点击“生成并部署到 Supabase”。

> 说明：`admin/` 页面包含配置管理与数据拼接混合功能，若你只关心数据拼接与部署，直接使用 `merger/index.html` 会更简单。

## 功能区说明

### 1. 公司配置

管理多租户公司信息，当前 company_id 决定操作的是哪家公司的数据。

| 字段/按钮 | 说明 |
|-----------|------|
| 公司名称 | 显示名称，如"演示公司" |
| 公司代码 | 系统内唯一标识（英文/数字，如 `demo-company`），一旦创建不建议修改 |
| 当前 company_id | 当前正在操作的公司，切换后所有配置和数据操作都指向该公司 |
| 配置版本 | 自动从后端填充，显示当前已发布版本号 |
| **公司列表** | 列出系统内所有公司，点击芯片可一键切换到该公司 |
| **创建公司** | 用"公司名称"和"公司代码"在系统中注册新公司 |

**常见操作**：
1. 在"公司代码"中填入目标公司 ID，其他区域的操作自动关联到该公司。
2. 点击"公司列表"可快速切换，无需手动填写代码。

---

### 2. 字段配置

定义报价台的数据字段：哪些字段参与搜索、哪些出现在复制结果、在结果卡中显示在哪个区域。

#### 字段表格说明

| 列名 | 说明 |
|------|------|
| key | 字段唯一标识，程序内部引用，建议用英文小写+下划线 |
| 显示名 | 界面展示的中文名称 |
| 类型 | `text`=文本，`number`=数字，`computed`=运行时计算 |
| 来源 | `price`=来自价格包，`stock`=来自库存包，`both`=两者均有，`computed`=实时计算 |
| Excel 别名 | 逗号分隔，merger 工具按此识别 Excel 列名（如"规格型号, 型号, 规格"） |
| 搜索 | 勾选后该字段参与关键词搜索 |
| 复制 | 勾选后该字段可出现在复制输出中（还需在"复制模板"中配置） |
| 显示区域 | 结果卡中的位置，见下方说明 |

#### 显示区域对照

| 区域值 | 卡片位置 | 典型字段 |
|--------|----------|----------|
| `identity` | 顶部标识，大字显示 | 规格型号 (spec)、编码 (code) |
| `metric` | 价格区，突出显示 | 面价 (face_price)、报价 (quote_price) |
| `chip` | 标签行，小徽章 | 特价活动 (special)、库存 (stock) |
| `detail` | 底部明细，小字 | 备注 (remark)、交期 (delivery) |

#### 常见操作
- 点击"**添加字段**"新增一行，填写各属性后保存草稿。
- 点击行末"删除"图标移除字段（注意：被规则或复制模板引用的字段删除后需一并更新）。

### 3. 报价规则

通过"条件 + 动作"的可视化构建器维护折扣规则。无需编写代码。

#### 规则结构

每条规则包含：
- **优先级**（数字越小越先执行）
- **条件**：满足条件时规则生效，支持多个条件（全部满足 = AND）
- **动作**：命中后执行的操作（如设置折扣）
- **默认规则标志**（`default=true`）：所有规则都不匹配时的兜底

#### 条件操作符

| 操作符 | 含义 |
|--------|------|
| `equals` | 字段值精确等于 |
| `contains` | 字段值包含（忽略大小写） |
| `regex` | 字段值匹配正则表达式 |
| `in` | 字段值在指定列表中 |
| `gt` / `gte` | 大于 / 大于等于 |
| `lt` / `lte` | 小于 / 小于等于 |
| `between` | 在数值范围内 |

#### 动作类型

| 动作 | 说明 |
|------|------|
| `set_discount` | 设置折扣百分比，如 `55` = 55 折 |
| `set_field` | 将某字段设置为指定值 |
| `set_formula` | 覆盖默认报价公式 |
| `stop` | 立即停止规则匹配 |

#### 示例规则配置

| 规则名 | 条件 | 折扣 |
|--------|------|------|
| EX 活动 | special 包含 "EX活动" | 32 折 |
| OSG 品牌 | brand 匹配正则 "OSG" | 36 折 |
| 默认 | （兜底规则，无条件） | 55 折 |

**注意**：必须有且只有一条 `default=true` 的规则，否则发布时校验会失败。

### 4. 复制模板

配置"复制勾选"功能的输出格式：哪些字段出现在复制结果、各字段的前缀、是否默认勾选。

#### 复制列属性

| 属性 | 说明 |
|------|------|
| 字段 key | 必须是"字段配置"中 `copyable=true` 的字段 |
| 显示名 | 复制面板中展示的列名 |
| 默认勾选 | 打开报价台时该列是否自动勾选 |
| 行类型 | `main`=主行（同行输出），`detail`=独立换行 |
| 前缀 | 输出时自动加在值前面（如"含税 "） |
| 后缀 | 输出时自动加在值后面 |

#### 输出格式示例

```
WNMG080408 含税86.4
交期 现货
```

- 第一行：所有 `line: "main"` 的勾选字段，空格拼接
- 以下各行：各个 `line: "detail"` 字段，每个独立一行

### 5. 页面显示

配置报价台的界面文案和结果卡布局。不需要修改前端代码。

| 配置项 | 说明 |
|--------|------|
| 应用标题 | 报价台页面标题，默认"智能询价系统" |
| 身份区 (identity) | 逗号分隔字段 key，控制卡片顶部显示的字段及顺序 |
| 指标区 (metrics) | 逗号分隔字段 key，控制价格区显示的字段及顺序 |
| 标签区 (chips) | 逗号分隔字段 key，控制标签行显示的字段及顺序 |
| 详情区 (details) | 逗号分隔字段 key，控制明细区显示的字段及顺序 |

#### 定价设置

| 配置项 | 说明 |
|--------|------|
| 默认公式 | 报价计算公式，支持字段 key 和内置函数（默认：`face_price * discount_percent / 100`） |
| 小数位 | 报价显示的小数位数（0=整数，1=一位小数） |
| 取整模式 | 超过整数阈值后的取整方式：`ceil`=向上取整，`floor`=向下取整，`round`=四舍五入 |
| 整数阈值 | 面价超过该值时报价取整为整数（默认 100） |

### 6. 数据拼接

整合原独立 merger 工具的功能，将品牌识别、数据合并、Bundle 生成与部署集成到配置中心，无需单独打开 merger 页面。

#### 阶段 1：品牌识别

上传多个品牌的原始 Excel 文件，系统根据文件名前缀自动识别品牌归属。

| 操作 | 说明 |
|------|------|
| 选择文件 | 支持 `.xlsx` / `.xls` / `.csv`，可多选 |
| **识别品牌** | 上传后按配置中 `merger.brand_rules` 的前缀规则自动匹配 |
| 修正品牌 | 检测结果中每行都有下拉框，可手动修正识别错误 |
| **按品牌写入数据** | 确认品牌归属后，一键将所有文件数据写入后端数据库 |
| 导入此文件 | 单独导入某个文件的数据（点击行末按钮） |

> **品牌识别规则**：在配置 JSON 中通过 `merger.brand_rules.brands` 定义品牌前缀。例如 `{"id": "MMC", "prefixes": ["MMC", "三菱"]}` 表示文件名以 "MMC" 或 "三菱" 开头的文件归属于 MMC 品牌。无法匹配的文件归入 `defaultBrand`。

#### 阶段 2：Bundle 生成与部署

从数据库当前数据生成价格包和库存包，供报价台前端加载使用。

| 操作 | 说明 |
|------|------|
| 价格包密码 | 填写则 AES-GCM 加密，留空则明文 base64 输出 |
| Supabase Anon Key | 部署时需要的 API 密钥，临时保存在浏览器会话（sessionStorage），关闭或刷新页面后将被清除 |
| **生成 Bundle** | 仅生成价格包 + 库存包，不部署（可在结果中预览数据量） |
| **生成并部署到 Supabase** | 生成后直接上传到 Supabase Storage，报价台即时可用 |

> **部署前提**：配置中必须包含 `data_source.base_url`（Supabase Storage 的 public URL）和 `data_source.price_bundle_file` / `data_source.stock_bundle_file`（文件名）。部署时会将 public URL 自动转换为写入 URL 并上传。

### 8. 发布配置

管理配置的导入/导出、校验和发布操作。

| 按钮 | 说明 |
|------|------|
| **导出 JSON** | 将当前配置下载为 `.json` 文件，用于备份或迁移 |
| **导出 YAML** | 将当前配置下载为 `.yaml` 文件，更易人工阅读 |
| **导入 JSON** | 上传一个 `.json` 配置文件，导入为草稿（不立即发布） |
| **校验配置** | 发布前检查配置合法性（默认规则、字段引用、公式安全等），发现问题会列出具体错误 |

高级 JSON 文本框展示当前完整配置，**只读展示**；如需用 JSON 直接编辑，用"导入 JSON"导入后再发布。

**发布流程**：
1. 在各功能区完成修改
2. 点击"**保存草稿**"（可选，用于中途暂存）
3. 点击"**发布配置**"发布为新版本
4. 右侧"实时预览"和"测试报价"可随时验证结果

---

### 9. 版本历史

查看所有历史配置版本并支持一键回滚。

点击"**加载历史**"加载版本列表，表格列说明：

| 列 | 说明 |
|----|------|
| 版本 | 配置 revision 标识（如 `2026-06-02.1`） |
| 状态 | `已发布`（当前使用）、`草稿`（未发布）、`已归档`（被新版本替代） |
| 发布时间 | 该版本发布的时间 |
| 创建时间 | 该版本创建的时间 |
| 操作 | 点击"**回滚**"将该版本重新发布为当前版本 |

**回滚说明**：
- 回滚会将选中的旧版本配置以新版本身份发布，不删除任何历史记录
- 回滚后，报价台下一次请求即使用该版本
- 审计日志会记录本次回滚操作

### 10. 审计日志

查看系统内所有配置操作的审计记录。

点击"**加载日志**"加载最近 50 条记录，表格列说明：

| 列 | 说明 |
|----|------|
| 时间 | 操作发生的 UTC 时间 |
| 操作 | 操作类型（如 `save_config`、`publish_config`、`replace_items`） |
| 对象类型 | 操作的对象类别（如 `config`、`items`） |
| 对象 ID | 操作的具体对象标识符 |
| 操作人 | 执行操作的用户（v1 单用户模式下为 `system`） |

> 审计日志按公司隔离，只显示当前 company_id 下的操作记录。

## 完整操作流程示例

### 场景：新增一个公司并配置首个报价规则

1. **创建公司**：在"公司配置"中填写公司名称和代码，点击"创建公司"。
2. **定义字段**：在"字段配置"中添加 `spec`（规格型号，搜索=是，复制=是，区域=identity）和 `face_price`（面价，类型=number，区域=metric）等字段。
3. **添加规则**：在"报价规则"中添加一条默认规则（`default=true`，折扣 55）。
4. **配置复制列**：在"复制模板"中添加 `spec`（主行）和 `quote_price`（主行，前缀"含税"）。
5. **发布**：顶栏点击"发布配置"，右侧"测试报价"输入关键词验证。

### 场景：修改折扣规则

1. 在"报价规则"中找到需要修改的规则，修改折扣百分比或条件。
2. 点击顶栏"**保存草稿**"暂存修改。
3. 在右侧"**测试报价**"中输入对应商品关键词，验证折扣计算结果是否正确。
4. 确认无误后点击"**发布配置**"。

### 场景：数据更新后回滚到旧版本

1. 发现新数据有问题，进入"版本历史"。
2. 找到最近的正确版本，点击"回滚"。
3. 报价台立即切换到该历史版本（无需重启服务）。

## 常见问题

**Q：为什么保存草稿后报价台没有变化？**  
A：草稿不影响线上运行，需要点击"**发布配置**"才会生效。

**Q：发布时提示校验失败怎么办？**  
常见原因：
- 缺少 `default=true` 的默认规则 → 在"报价规则"中添加一条默认规则
- 规则中的字段 key 不存在 → 检查条件中引用的字段是否已在"字段配置"中定义
- 公式包含不允许的函数 → 报价公式只允许使用 `ceil`/`floor`/`round`/`min`/`max` 和字段 key

**Q：如何备份当前配置？**  
在"发布配置"中点击"导出 JSON"或"导出 YAML"下载备份文件。也可以通过"版本历史"查看所有历史版本。

**Q：多家公司的数据会互相影响吗？**  
不会。所有数据（配置、商品、规则、审计日志）都按 `company_id` 完全隔离，公司 A 无法读取公司 B 的任何信息。
