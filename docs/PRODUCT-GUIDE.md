## 一、产品定位

### 1.1 产品名称

智能询价系统（Smart Quotation）

### 1.2 一句话定位

面向 B2B 刀具/工具行业的**多租户配置驱动报价平台**——管理员零代码配置价格规则，销售/客户实时查询报价与库存。

### 1.3 解决的核心问题

| 痛点 | 传统方式 | 本系统方案 |
|------|----------|------------|
| 价格更新慢 | Excel 邮件下发，客户拿到的报价滞后 | 配置中心发布后即时生效，前端缓存自动更新 |
| 折扣管理乱 | 每个销售记不同的折扣表 | 配置驱动，品牌×活动规则自动匹配折扣 |
| 面价泄露风险 | 报价单含面价，客户可推算折扣 | 公司角色脱敏——只看到最终报价，看不到面价和折扣率 |
| 多客户隔离难 | 一个 Excel 管所有客户，容易发错 | 多租户 `company_id` 隔离，每家客户独立配置/数据/利润率 |
| 库存查询繁琐 | 手动登录三菱官网逐条查 | 一键批量查询，结果直接进剪贴板 |

### 1.4 目标用户

| 角色 | 使用场景 | 核心诉求 |
|------|----------|----------|
| **系统管理员** | 配置中心（admin） | 零代码管理字段/规则/数据/发布 |
| **销售/内部员工** | 报价台（apps），admin 模式 | 看面价+折扣，手动调折扣，快速报价 |
| **公司客户** | 报价台（apps），company 模式 | 看脱敏报价（含利润率），调利润率，查库存 |
| **库存查询用户** | 报价台（apps），stock_only 模式 | 仅查三菱实时库存，不看任何价格 |

## 二、核心价值主张

### 2.1 对管理员

- **零代码配置**：所有字段、规则、布局在浏览器 GUI 中完成，无需改代码
- **版本管理**：每次发布自动归档，支持一键回滚
- **审计追踪**：所有配置变更记录在案，按公司隔离
- **一键部署**：发布配置时自动同步到 Supabase，前端即时生效

### 2.2 对销售（admin 模式）

- **批量查询**：多行输入，一次查多个规格型号
- **灵活调价**：折扣步进按钮（长按连续调整）+ 手动输入
- **含税/未税切换**：一键切换，每步独立取整
- **复制即用**：勾选结果后一键复制，格式可配置

### 2.3 对公司客户（company 模式）

- **脱敏报价**：看不到面价和折扣率，只看到含利润率的最终报价
- **自调利润率**：可在管理员设定的范围内调整利润率
- **安全访问**：令牌通过安全链接发放，页签关闭即失效
- **移动端友好**：响应式布局，手机/平板可用

### 2.4 对库存查询用户

- **三菱官网直连**：实时库存（上海仓 + 日本仓）
- **批量查询**：单次最多 50 条，结果自动进剪贴板
- **频率保护**：60 秒内最多 30 次，防滥用

## 三、用户角色与权限矩阵

### 3.1 角色定义

| 角色 | 认证方式 | 数据可见性 | 操作权限 |
|------|----------|------------|----------|
| admin | Admin API Key（Bearer） | 全部数据（面价/折扣/报价） | 调折扣、查库存、看完整数据 |
| company | 公司访问令牌（X-Company-Token） | 脱敏数据（无面价/无折扣规则） | 调利润率、查库存、看报价 |
| stock_only | 库存查询密钥（X-Stock-Key） | 无价格数据 | 仅查三菱库存 |

### 3.2 数据脱敏对照

| 数据项 | admin | company | stock_only |
|--------|-------|---------|------------|
| 面价 (face_price) | ✅ 可见 | ❌ 脱敏移除 | ❌ 不加载 |
| 折扣率 (discount_percent) | ✅ 可见 | ❌ 脱敏移除 | ❌ 不加载 |
| 折扣规则 (rules/discount_rules) | ✅ 配置含 | ❌ 配置脱敏 | ❌ 不加载 |
| 定价公式 (default_formula) | ✅ 可见 | ❌ 脱敏移除 | ❌ 不加载 |
| 预计算报价 (quote_price) | ✅ 前端计算 | ✅ 服务端预计算 | ❌ 不加载 |
| 库存 (stock) | ✅ 可见 | ✅ 可见 | ✅ 可见 |
| 规格型号/编码等 | ✅ 可见 | ✅ 可见 | ✅ 可见 |

### 3.3 定价公式

| 模式 | 含税价 | 未税价 |
|------|--------|--------|
| admin | `rounding(面价 × 折扣%)` | `rounding(含税价 / (1 + 税率%))` |
| company | `rounding(服务端报价 × (1 + 利润率%))` | `rounding(含利润含税价 / (1 + 税率%))` |

> **设计要点**：
> - 税率为全局配置（`config.pricing.tax_rate`，默认 13%），不再按公司配置。中国工业品增值税率几乎统一为 13%，按公司配置会造成逻辑错位。
> - 面价含税属性由 `config.pricing.face_price_tax_inclusive` 标注（默认含税）。上传未税价格表时，系统自动 ×(1+税率) 转为含税存储，避免未税价被误当含税价处理。
> - company 模式的 `quote_price` 由服务端预计算（面价 × 品牌折扣），前端在此基础上叠加利润率和税务。客户端永远接触不到面价和折扣率原始值。

## 四、核心使用流程

### 4.1 管理员流程：首次配置

```
启动后端 → 登录 admin → 创建公司 → 配置字段 → 配置规则
→ 配置复制模板 → 配置页面显示 → 上传数据 → 发布配置 → 生成客户链接
```

**关键步骤说明**：

1. **创建公司**：填写公司 ID（英文）、名称、利润率（税率已移至全局「定价设置」）
2. **配置字段**：定义哪些字段参与搜索、显示在结果卡的哪个区域
3. **配置规则**：条件 → 折扣（必须有且只有一条 `default=true` 的兜底规则）
4. **上传数据**：在「数据拼接区」上传 Excel，系统按文件名自动识别品牌
5. **发布配置**：点击「发布配置」→ 自动同步到 Supabase → 前端即时可用
6. **生成客户链接**：在公司管理中「重新生成令牌」→ 链接自动复制到剪贴板

### 4.2 管理员流程：日常数据更新

```
上传新 Excel → 写入数据库 → 生成并部署 Bundle → 前端自动刷新
```

> 只改数据时不需要重新发布配置。前端通过 `version.json` 检测数据版本变化，自动重新下载 Bundle。

### 4.3 客户流程：首次使用

```
收到管理员发的链接 → 浏览器打开 → authGate 自动填充令牌 → 进入报价台
```

**链接格式**（安全版）：
```
https://你的域名/#company_id=acme&token=xxxxxxxx
```

> 令牌通过 URL fragment（`#` 后）传递，不会发送到服务器，不会出现在访问日志中。读取后自动从地址栏清除；默认存入 `localStorage` 保持登录（关闭浏览器再打开免输链接），公用电脑可在登录页取消「保持登录」勾选退回 `sessionStorage`。页头「退出」按钮可一键清除全部本地凭证。

### 4.4 客户流程：日常使用

```
输入规格型号 → 点击「智能查询」→ 查看报价 → 调利润率（可选）→ 勾选 → 复制
```

**库存查询**：
- 点击「三菱库存」→ 批量查询选中项的实时库存
- 结果自动进剪贴板，可直接粘贴给客户

### 5.2 当前版本不支持

| 功能 | 原因 | 替代方案 |
|------|------|----------|
| 客户服务端登录 | 路线图 P2 | 令牌认证（默认 localStorage 持久，可切 session） |
| 多租户并发写入 | SQLite 单写者 | 低并发 B2B 场景可接受 |
| PostgreSQL | 路线图 P2 | SQLite WAL 模式 |
| 部署文档 | 路线图 P2 | README + _DEPLOYMENT-STEPS.md（本地）+ gui-admin-guide |

### 5.3 认证模式说明

> **产品边界**：当前认证为前端本地模式，profile 随「保持登录」偏好存于 `localStorage`（默认）或 `sessionStorage`。
>
> 如需真正的多租户客户登录（服务端校验、密码哈希、会话令牌），需在 backend 补全 `customers` / `customer_sessions` 表与相关 API 端点（见路线图 P2）。
>
> 当前模式下，公司访问令牌由管理员生成并通过安全渠道（邮件/微信）分享给客户。令牌有 90 天有效期，可随时由管理员重新生成（旧令牌立即失效）。

## 六、安全设计摘要

### 6.1 认证安全

- **Admin API Key**：至少 16 字符，`secrets.compare_digest` 防时序攻击
- **公司访问令牌**：256 位熵（`secrets.token_urlsafe(32)`），90 天过期
- **库存查询密钥**：独立于 Admin Key，防提权
- **令牌传输**：仅通过 HTTP 头（`X-Company-Token`），不走 URL query
- **令牌存储**：默认 `localStorage` 持久化（保持登录），登录页可取消勾选退回 `sessionStorage`；admin 令牌（无 company_id）始终仅存 `sessionStorage`；401 时自动清除全部凭证并提示重新索取链接

### 6.2 数据安全

- **脱敏**：company 角色看不到面价、折扣率、定价公式、折扣规则
- **部署防护**：Supabase 部署时强制从数据库重建脱敏 bundle，不信任客户端输入
- **SSRF 防护**：Supabase URL 校验（HTTPS + 域名后缀 + 内网 IP 检查）
- **公式安全**：AST 白名单评估器，拒绝 `__import__` 等危险调用

### 6.3 防爆破

- **认证频率限制**：SQLite 持久化，跨 Worker 共享
- **库存查询限制**：60s/30 次 + 单次 50 条上限

### 6.4 传输安全

- **CSP**：`script-src 'self' https://browser.sentry-cdn.com`（SheetJS 已自托管，仅保留 Sentry SDK CDN 白名单）；`connect-src` 白名单限定 Supabase/Sentry/Railway/Render 域名（已移除 `https:` 通配防 XSS 外泄）
- **HSTS**：`Strict-Transport-Security` 强制 HTTPS
- **frame-ancestors 'none'**：防点击劫持

## 八、部署架构

### 8.1 架构图

```
┌─────────────────────────────────────────────────────┐
│  admin/ (浏览器端 GUI 配置中心)                     │
│  字段配置 · 报价规则 · 数据拼接 · Bundle 生成 · 发布 │
└────────────────────┬────────────────────────────────┘
                     │ POST /api/* (Bearer Admin Key)
                     ▼
┌─────────────────────────────────────────────────────┐
│  backend/ (FastAPI + SQLite)                         │
│  多租户 company_id 隔离 · CRUD · 审计 · Bundle 部署  │
└────────────────────┬────────────────────────────────┘
                     │ 部署 Bundle (脱敏 config + price + stock)
                     ▼
┌─────────────────────────────────────────────────────┐
│  Supabase Storage (公开桶)                          │
│  config.json · price.bundle.json · stock.bundle.json│
└────────────────────┬────────────────────────────────┘
                     │ fetch + Cache API (X-Company-Token)
                     ▼
┌─────────────────────────────────────────────────────┐
│  apps/ (静态前端报价台, Netlify)                     │
│  模糊查询 · 折扣/利润计算 · 含税/未税切换 · 复制     │
│  三菱库存实时查询（POST /api/stock-query）           │
└─────────────────────────────────────────────────────┘
```

### 8.2 部署组件

| 组件 | 部署位置 | 用途 |
|------|----------|------|
| 前端报价台 (apps/) | Netlify | 客户访问入口 |
| 后端 API (backend/) | Railway / Render | 配置管理 + 数据 CRUD + 库存查询代理 |
| 数据存储 (Supabase) | Supabase Storage | 公开桶存放脱敏 Bundle |
| 数据库 (SQLite) | 后端同机 | 配置 + 商品数据 + 审计日志（WAL 模式） |

### 8.3 环境变量速览

| 变量 | 必填 | 说明 |
|------|------|------|
| `ADMIN_API_KEY` | ✅ | Admin 后台密钥（≥16 字符） |
| `STOCK_QUERY_KEY` | 生产必填 | 三菱库存查询专用密钥（独立于 Admin Key） |
| `MMC_USERNAME` / `MMC_PASSWORD` | 可选 | 三菱官网凭据 |
| `SQ_SUPABASE_BASE_URL` | 可选 | Supabase Storage 地址（也可在 admin 配置） |
| `SQ_PROD_API_BASE` | 可选 | 生产后端 API 地址（前端也可通过 `?api=` 注入） |
| `ALLOW_ORIGINS` | 生产 | CORS 允许来源 |
| `SQ_DEV` | 本地开发 | 设为 `1` 跳过密钥强校验 |

| Bundle | 数据包（price.bundle.json / stock.bundle.json），前端通过 fetch 加载 |
| company_id | 多租户隔离单位，所有数据按此过滤 |
| authGate | 前端认证覆盖层，未认证时显示输入框 |
| 脱敏 | 移除敏感字段（面价、折扣率），仅保留预计算报价 |
| 取整阈值 | 面价超过该值时报价取整为整数（默认 100） |
| 步进 | 折扣/利润率的调整步长（默认 0.1 个百分点） |
| GWT-RPC | 三菱官网使用的远程调用协议，后端通过它查询库存 |
| WAL | SQLite Write-Ahead Logging，提升并发读写性能 |
